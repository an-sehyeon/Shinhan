3. Spring MVC
    - Restful API(Representational State Transfer)
        - HTTP와 URL 기반으로 자원에 접근할 수 있도록 제공하는 어플리케이션 개발 인터페이스
        - 자원에 대한 행위는 HTTP Method(GET, POST, PUT, DELETE)를 통해 표현
        - 해당 resource에 대한 CRUD operation을 적용하는 것
        - REST API의 설계 규칙을 올바르게 지킨 시스템을 "RESTful하다"라고 함.

        - Rest API 세부규칙
            - 슬래시(/)는 계층관계를 나타낸다.
            - URI 마지막에 슬래시를 포함하지 않는다.
            - 경로는 소문자를 사용한다.
            - 경로는 언더바를 사용하지 않고 하이픈(-)을 사용한다.

        - JSON 기능 이용(의존성 주입)
        <dependency>
            <groupId>com.fasterxml.jackson.core</groupId>
            <artifactId>jackson-databind</artifactId>
            <version>2.12.2</version>
        </dependency>

        - JSON 사용하는 이유
            - XML기반의 다른 SOAP(Simple Object Access Protocol)등 다른 프로토콜에 비해 매우 가볍고 적은 text로 데이터 표현이 가능
            - JSON은 자바스크립트에서 사용되는 기본 직렬화 프로토콜이다.
